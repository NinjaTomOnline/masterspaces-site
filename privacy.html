Privacy

Local window data, no analytics in the current release candidate.

MasterSpaces uses macOS Accessibility only to read and move windows you control. The current release candidate does not include accounts, analytics, advertising SDKs, remote sync, or AI processing.

What stays on your Mac

Saved layouts, workspace resources, restore variants, preferences, dismissed tips, and quick-start progress are stored locally using app storage on your Mac. You can export and import workspace archives manually when you choose.

Accessibility permission

MasterSpaces asks for Accessibility permission so it can inspect the frontmost window, read window frames, and move or resize windows for snapping and layout restore. This permission is required for core functionality and is explained during onboarding.

Workspace resources

If you add apps, files, websites, or scripts to a workspace, MasterSpaces stores the references needed to launch them later. It does not upload those resources or their contents.

Debug logs and release QA evidence

Debug builds can write local display and drag logs for troubleshooting. Release QA artifacts are generated by local scripts and are not transmitted by the app. If you share a debug artifact with support, review it first for file paths or display details you consider private.

What is not included today

The current release candidate does not ship cloud sync, native macOS Spaces management, or the planned AI assistant. If those features are added later, this policy should be updated before public release.

Contact

Privacy questions can be sent to privacy@masterspaces.app.
